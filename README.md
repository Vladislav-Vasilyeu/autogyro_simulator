# Симулятор управления автожиром М-24 «Орион»

Курсовой проект по дисциплине **«Компьютерные мультимедийные системы»**.

Проект представляет собой мультимедийный симулятор управления автожиром на базе Unity. В репозитории собраны исходный Unity-проект, 3D-модель автожира, материалы для моделирования, веб-оболочка с теоретическими разделами, интерактивная озвученная карта узлов автожира, WebGL-сборка и Windows-сборка симулятора.

## О проекте

Цель проекта — создать учебный интерактивный симулятор, который помогает познакомиться с устройством автожира М-24 «Орион», принципами его работы и базовыми действиями пилота при запуске, рулении, раскрутке ротора, взлете и прохождении маршрута.

В проекте объединены несколько мультимедийных компонентов:

- 3D-сцена автожира и окружения в Unity;
- пользовательская физическая модель полета автожира;
- интерактивная кабина с тумблерами и подсказками;
- телеметрия полета и состояния бортовых систем;
- учебный режим с последовательными практическими заданиями;
- таблица результатов тренировочного полета;
- веб-страницы с теорией, описанием конструкции и встроенным Unity WebGL-симулятором;
- интерактивная озвученная карта частей автожира;
- исходные материалы 3D-моделирования и изображения, использованные при подготовке проекта.

## Основные возможности

- **Физика полета автожира**: тяга маршевого винта, подъемная сила несущего ротора, авторотация, сопротивление воздуха, ограничение скорости, поведение на земле и в воздухе.
- **Запуск двигателя и бортовых систем**: мастер-питание, батарея, генератор, EMS, топливные насосы, зажигание и кнопка стартера.
- **Интерактивная кабина**: переключатели можно нажимать мышью, а при удержании `Left Alt` отображаются описания элементов.
- **Камеры**: переключение между внешним видом и видом из кабины.
- **Телеметрия**: высота, скорость, обороты ротора, обороты двигателя и состояние систем кабины.
- **Практический сценарий**: запуск, руление, предварительная раскрутка ротора, взлет, прохождение контрольных точек.
- **Фиксация результата**: запись оборотов ротора, скорости взлета, максимальной высоты и времени выполнения.
- **Веб-часть**: титульная страница, теория, описание автожира, принцип работы, встроенный WebGL-симулятор.
- **Озвученная интерактивная карта**: отдельный мультимедийный модуль с частями автожира и аудиоописаниями.

## Технологии

## Ключевые скрипты Unity

Основная логика находится в `Unity/Project/Autogyro/Assets/Scripts/`.

| Скрипт | Назначение |
| --- | --- |
| `GyrocopterFlight.cs` | Физика движения автожира, тяга, подъемная сила, авторотация, газ, тормоз, восстановление положения и вращение винтов. |
| `AviationSystems.cs` | Логика бортового питания, двигателя Rotax, оборотов двигателя и вывода телеметрии. |
| `CockpitSwitch.cs` | Интерактивный тумблер кабины с состоянием и текстовым описанием. |
| `CockpitInteractor.cs` | Луч из камеры кабины для нажатия на элементы и показа подсказок. |
| `CameraSwitcher.cs` | Переключение между внешней камерой и камерой кабины. |
| `PracticeManager.cs` | Учебный сценарий: запуск, руление, раскрутка, взлет, чекпоинты и итог. |
| `TableManager.cs` | Запись результатов тренировочных циклов в UI-таблицу. |
| `UIController.cs` | Управление интерфейсными элементами. |
| `DoorController.cs` | Анимация/управление дверью. |
| `StarterTrigger.cs` | Запуск двигателя через интерактивный элемент. |
| `JoystickController.cs` | Визуальное/логическое управление джойстиком. |
| `HighlightElement.cs` | Подсветка интерактивных элементов. |

## Управление в симуляторе

| Действие | Клавиша / ввод |
| --- | --- |
| Газ | Колесико мыши |

| Стояночный тормоз | `B` |

| Восстановление положения автожира | `R` |
| Подсказки элементов кабины | Удерживать `Left Alt` и навести курсор |

## Как запустить Unity-проект

1. Установите **Unity 6000.4.1f1** или совместимую версию Unity 6.
2. Откройте Unity Hub.
3. Выберите **Add project from disk**.
4. Укажите папку:

```text
Unity/Project/Autogyro
```

5. Дождитесь импорта зависимостей и открытия проекта.
6. Откройте сцену:

```text
Assets/Scenes/SampleScene.unity
```

7. Нажмите **Play**.

## Как запустить готовую Windows-сборку

Windows-сборка находится в папке:

```text
build/
```

Для запуска откройте:

```text
build/Autogyro.exe
```

Важно: рядом с `Autogyro.exe` должны оставаться папки `Autogyro_Data`, `MonoBleedingEdge`, `D3D12` и библиотеки Unity. Без них сборка не запустится.

## Как открыть веб-версию

Веб-часть находится в папке:

```text
web/
```

Главная страница:

```text
web/index.html
```

Страница со встроенным симулятором:

```text
web/simulator.html
```

Для корректной работы WebGL-сборки лучше запускать сайт через локальный сервер, а не простым открытием HTML-файла. Например:

```bash
cd web
python -m http.server 8000
```

После этого откройте в браузере:

```text
http://localhost:8000
```

## WebGL-сборка

В проекте есть две WebGL-сборки:

- `web/web_build/` — сборка, встроенная в страницу `web/simulator.html`;
- `BuildWeb/` — отдельная WebGL-сборка с собственным `index.html`.

Если WebGL не запускается при открытии файла напрямую, используйте локальный HTTP-сервер. Это стандартное ограничение браузеров для Unity WebGL.

## 3D-модель и материалы

## Рекомендации перед публикацией на GitHub

Unity-проекты быстро становятся тяжелыми, поэтому перед публикацией желательно проверить состав репозитория.

Обычно **не стоит загружать**:

- `Unity/Project/Autogyro/Library/`
- `Unity/Project/Autogyro/Logs/`
- `Unity/Project/Autogyro/obj/`
- `Unity/Project/Autogyro/.vs/`
- временные архивы `*.zip`, `*.rar`
- папки `*_BurstDebugInformation_DoNotShip`
- пользовательские настройки IDE и Unity, если они не нужны для проверки проекта

Можно оставить в репозитории:

- `Unity/Project/Autogyro/Assets/`
- `Unity/Project/Autogyro/Packages/`
- `Unity/Project/Autogyro/ProjectSettings/`
- `web/`
- `model/`
- исходники 3D-моделей и референсы, если размер репозитория допустим
- готовые сборки, если преподавателю или проверяющему важно запускать проект без Unity

Если файлы модели, видео или сборок превышают лимиты GitHub, их лучше перенести в **Git LFS** или приложить как архив в разделе **Releases**.
