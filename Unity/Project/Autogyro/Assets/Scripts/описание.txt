Этот скрипт написан с использованием современных стандартов **Unity 6** (о чем свидетельствуют свойства `linearVelocity`, `linearDamping` и `angularDamping`, пришедшие на замену устаревшим `velocity`, `drag` и `angularDrag`).

Главная техническая особенность твоей 3D-модели, под которую адаптирован этот код: **осью движения (носом автожира) является локальная ось X (красная)**, а не стандартная для Unity ось Z (синяя). Соответственно, боковой осью является ось Z (`transform.forward`).

Ниже представлен максимально подробный академический и технический разбор всего файла — от каждой переменной до каждой физической формулы.

## 1. Подробный разбор всех переменных (Полей класса)

### Ссылки на компоненты (`[Header("Ссылки на компоненты")]`)

* `public AviationSystems aviationSystems;` — ссылка на сторонний скрипт, управляющий логикой двигателя (запущен ли, текущие и максимальные обороты). Передает в наш скрипт состояние мотора.
* `public Transform mainRotor;` — ссылка на объект несущего винта (ротора). Используется для визуального вращения лопастей и наклона всего диска винта.
* `public Transform pusherPropeller;` — ссылка на маршевый (толкающий) винт, расположенный сзади. Используется для визуального вращения.
* `public Transform rudderKiel;` — ссылка на руль направления (киль) на хвосте. Поворачивается влево/вправо при рулении.

### Звуковое сопровождение (`[Header("Звуковое сопровождение двигателя")]`)

* `public AudioSource startAudioSource;` — компонент для воспроизведения звука стартера/запуска (играет один раз).
* `public AudioSource loopAudioSource;` — компонент для постоянного зацикленного гула работающего мотора.
* `public AudioClip engineStartClip;` — аудиофайл с записью запуска двигателя.
* `public AudioClip engineLoopClip;` — аудиофайл с зацикленной записью работы.

### Лётная аэродинамика (`[Header("Лётная аэродинамика (...)")]`)

* `public float maxEngineThrust = 5800f;` — максимальная сила тяги маршевого винта в Ньютонах. Прикладывается по оси X (`transform.right`).
* `public float liftCoefficient = 3.5f;` — коэффициент, определяющий, сколько подъемной силы генерируют лопасти ротора при определенных оборотах.
* `public float airDrag = 0.28f;` — базовое лобовое сопротивление воздуха для фюзеляжа автожира во время полета.

### Настройки чувствительности (`[Header("Настройки чувствительности...")]`)

* `public float pitchSensitivity = 4.0f;` — множитель скорости наклона автожира по тангажу (нос вверх/вниз) при движении мыши.
* `public float rollSensitivity = 4.5f;` — множитель скорости наклона по крену (влево/вправо) при движении мыши.
* `public float yawSensitivity = 5.0f;` — множитель скорости поворота по курсу (рыскание) при нажатии клавиш `A/D`.

### Пределы наклона ротора (`[Header("Пределы механического наклона...")]`)

* `public float maxRotorTiltPitch = 12f;` — максимальный угол (в градусах), на который втулка ротора физически может наклониться вперед или назад.
* `public float maxRotorTiltRoll = 9f;` — максимальный угол (в градусах) наклона втулки ротора влево или вправо.

### Телеметрия состояния (`[Header("Телеметрия состояния")]`)

* `public float forwardSpeed = 0f;` — текущая скорость движения вперед вдоль оси X (в метрах в секунду). Выводится в реальном времени.
* `public float rotorRPM = 0f;` — текущие обороты несущего винта в минуту (в реальном времени).
* `public float throttle = 0f;` — значение положения ручки газа от `0.0` (холостой ход) до `1.0` (полный газ).
* `public bool isParkingBrakeOn = true;` — флаг стояночного тормоза. Если `true`, колеса заблокированы.

### Внутренние приватные переменные

* `currentRotorTiltPitch` / `currentRotorTiltRoll` — текущие физические углы наклона втулки ротора в данный момент времени (плавно стремятся к командам пилота).
* `currentRotorVisualAngle` — накопительный угол вращения лопастей ротора (используется для визуализации вращения).
* `rb` — кэшированная ссылка на компонент `Rigidbody` (физическое тело автожира).
* `isGrounded` — флаг, находится ли автожир на земле (`true`) или в воздухе (`false`).
* `wasEngineRunningLastFrame` — технический флаг для отслеживания изменения состояния двигателя (нужен, чтобы поймать момент, когда двигатель *только что* завелся).
* `starterFinished` — флаг, завершился ли звук запуска мотора, чтобы переключиться на циклический гул.

## 2. Построчный разбор методов и алгоритмов

### Метод `Start()`

Вызывается один раз при старте сцены. Инициализирует начальные физические параметры.

* `rb = GetComponent<Rigidbody>();` — захватывает компонент физики с текущего объекта.
* `rb.mass = 450f;` — принудительно выставляет массу автожира в $450\text{ кг}$ (реальный вес пустого Magni M-24 с пилотами).
* `rb.useGravity = true; rb.isKinematic = false;` — включает гравитацию и физическое взаимодействие.
* `rb.linearDamping = 0.02f;` — сопротивление линейной скорости (очень малое, так как сопротивление воздуха мы считаем вручную аэродинамической формулой).
* `rb.angularDamping = 2.0f;` — угловое сопротивление, предотвращает бесконечное хаотичное вращение («волчок») и стабилизирует машину.
* `isParkingBrakeOn = true;` — на старте автожир зафиксирован на тормозах.
* `wasEngineRunningLastFrame = aviationSystems.isEngineRunning;` — запоминает начальное состояние двигателя.

---

### Метод `Update()`

Вызывается каждый кадр. Отвечает за опрос ввода пользователя, логику звуков и механику авторотации винта.

1. **Проверка земли:**
`isGrounded = Physics.Raycast(transform.position, -transform.up, 1.65f);`
Пускает невидимый луч из центра автожира вертикально вниз (`-transform.up`) на дистанцию $1.65\text{ метра}$. Если луч упирается в поверхность (полосу), значит мы на земле.
2. **Стояночный тормоз:**
`if (Input.GetKeyDown(KeyCode.B) && isGrounded) { isParkingBrakeOn = !isParkingBrakeOn; }`
При нажатии на клавишу `B` на земле тормоз переключается (включается/выключается).
3. **Вызов сопутствующих методов:**
`HandleRecovery();` — опрос клавиши эвакуации `R`.
`HandleEngineAudio();` — обновление звукового процессора.
4. **Проверка работы двигателя:**
Если `aviationSystems.isEngineRunning == false`, то ручка газа принудительно сбрасывается в 0, а обороты ротора (`rotorRPM`) плавно падают до нуля со скоростью `15 * Time.deltaTime`. Метод прерывается (`return`).
5. **Управление газом (Throttle):**
Если не зажат `LeftAlt` (который нужен для свободного обзора мышью), скролл колесика мыши изменяет значение газа: `throttle += scroll * 2.0f;` с ограничением в диапазоне `[0, 1]` с помощью `Mathf.Clamp01`.
6. **ФОРМУЛА АВТОРОТАЦИИ (Как раскручивается верхний винт):**
У автожира верхний винт вращается сам от набегающего потока воздуха.
* Сначала считается общая горизонтальная скорость движения:
`float currentSidewaysSpeed = Vector3.Dot(rb.linearVelocity, transform.forward);`
`float totalHorizontalSpeed = Mathf.Sqrt(forwardSpeed * forwardSpeed + currentSidewaysSpeed * currentSidewaysSpeed);`
* **Режим предварительной раскрутки на земле (Pre-spin):**
Если зажат `Space` на земле, винт механически подключается к двигателю для начального старта:
`float enginePreSpinDest = Mathf.Lerp(0f, 290f, aviationSystems.engineRPM / aviationSystems.maxRPM);`
Винт плавно раскручивается максимум до $290\text{ оборотов в минуту}$ в зависимости от оборотов мотора.
* **Режим авторотации в полете (Основная формула):**
Если пробег по полосе или полет, то обороты зависят от физики воздуха:
`float verticalAirflow = Vector3.Dot(rb.linearVelocity, transform.up);` (вертикальный обдув винта).
`float tiltFactor = Mathf.Clamp01(currentRotorTiltPitch / maxRotorTiltPitch);` (чем сильнее наклонен ротор назад, тем больше воздуха в него попадает).
`float autorotationDrive = (totalHorizontalSpeed * 9.8f) * (1.0f + tiltFactor * 0.5f) - (verticalAirflow * 4.5f);`
*Смысл формулы:* Горизонтальная скорость умножается на гравитационную константу и увеличивается до +50%, если ротор наклонен назад. Вертикальное снижение (отрицательный `verticalAirflow`) раскручивает винт еще сильнее (минус на минус дает плюс).
* **Защита от падения оборотов в воздухе:**
`if (!isGrounded && totalHorizontalSpeed > 12f && targetRotorRPM < 200f) { targetRotorRPM = 210f; }`
Если мы летим со скоростью выше $12\text{ м/с}$ и расчетные обороты упали ниже критических, код искусственно удерживает их на уровне $210\text{ RPM}$, чтобы игрок не провалился из-за математической погрешности симуляции Unity.
* Применение оборотов: `rotorRPM = Mathf.MoveTowards(rotorRPM, targetRotorRPM, Time.deltaTime * 16f);` (плавное изменение).

### Метод `FixedUpdate()`

Вызывается с фиксированной частотой (обычно 50 раз в секунду). Здесь происходит расчет всех физических сил и крутящих моментов.

1. **Расчет векторов скоростей:**
* `forwardSpeed = Vector3.Dot(rb.linearVelocity, transform.right);` — проецирует общую скорость физического тела на локальную ось X (движение вперед). Если значение отрицательное (катимся назад), сбрасывается в 0.
* `sidewaysSpeed = Vector3.Dot(rb.linearVelocity, transform.forward);` — скорость бокового сноса (ось Z).


2. **Аэродинамическое сопротивление фюзеляжа:**
* `float rotorDragModifier = 1.0f + (currentRotorTiltPitch > 0 ? (currentRotorTiltPitch / maxRotorTiltPitch) * 1.2f : 0f);` — если ротор наклонен назад, он работает как огромный воздушный тормоз (парашют), увеличивая сопротивление воздуха до +120%.
* `float currentAirDrag = isGrounded ? Mathf.Lerp(0.01f, airDrag, forwardSpeed / 14f) : airDrag * rotorDragModifier;` — на земле сопротивление минимально, в воздухе — полноценное, с учетом наклона винта.
* `float dragForceValue = 0.5f * currentAirDrag * forwardSpeed * forwardSpeed;` — классическая ньютоновская формула лобового сопротивления: $F = 0.5 \cdot \rho \cdot C_d \cdot v^2$.
* `rb.AddForce(-transform.right * dragForceValue, ForceMode.Force);` — прикладывает эту силу против хода движения (тормозит автожир).


3. **Ограничение максимальной скорости:**
Если `forwardSpeed` превышает $44.4\text{ м/с}$ (~$160\text{ км/ч}$ — летно-технический предел Magni M-24), то вектор скорости жестко усекается, чтобы модель не улетала в космос.
4. **Работа стояночного тормоза:**
Если тормоз включен на земле, к машине применяется колоссальное искусственное затухание скорости: `rb.AddForce(-rb.linearVelocity * 35f, ForceMode.Acceleration);`.
5. **Считывание команд управления (РУС и Педали):**
Если зажата левая кнопка мыши (`Input.GetMouseButton(0)`), то перемещение мыши по вертикали (`Mouse Y`) записывается в `pitchInput`, по горизонтали (`Mouse X`) — в `rollInput`. Клавиши `A/D` задают `yawInput` (педали).
6. **Расчет наклона втулки ротора:**
`currentRotorTiltPitch = Mathf.MoveTowards(currentRotorTiltPitch, pitchInput * maxRotorTiltPitch, Time.fixedDeltaTime * 22f);`
Каждый физический шаг углы наклона втулки плавно движутся к целевым со скоростью 22 градуса в секунду.
Затем вычисляется нормаль вектора тяги ротора (`rotorNormal`): базовый вектор «вверх» (`transform.up`) наклоняется матрицами кватернионов в соответствии с текущими углами втулки.
7. **РАСЧЕТ ПОДЪЕМНОЙ СИЛЫ ВИНТА:**
```csharp
float speedFactor = Mathf.Clamp(totalHorizontalSpeed / 10f, 0.15f, 2.4f);
float lift = (rotorRPM * rotorRPM) * liftCoefficient * 0.0055f * speedFactor;

```


*Формула подъемной силы:* Зависит **квадратично** от оборотов ротора (`rotorRPM * rotorRPM`), умножается на коэффициент подъемной силы и на коэффициент скорости `speedFactor` (чем быстрее автожир бежит по полосе, тем эффективнее набегающий поток воздуха создает подъемную силу на лопастях).
* **Эффект экрана (Ground Effect):**
`if (groundDistance < 5.0f) { lift *= Mathf.Lerp(1.25f, 1.0f, groundDistance / 5.0f); }`
Если автожир находится у самой земли (ниже $5\text{ метров}$), подъемная сила увеличивается до +25% за счет сжатия под крылом «воздушной подушки», как у реального вертолета или самолета.
* Сила прикладывается в физический движок: `rb.AddForce(rotorNormal * lift, ForceMode.Force);`.
* **Эффект маятника (Pendulum Stability):**
Так как кабина автожира подвешена под ротором на жесткой мачте, при наклоне ротора возникает возвращающий маятниковый момент сил, который пытается выровнять тяжелую кабину вслед за винтом. Он рассчитывается и прикладывается через крутящие моменты `rb.AddTorque`.


8. **Расчет продольной тяги маршевого двигателя:**
`float currentThrust = throttle * maxEngineThrust * thrustModifier;`
Сила тяги зависит от ручки газа, максимальной тяги ($5800\text{ Н}$) и оборотов двигателя из внешнего скрипта. Прикладывается вперед по ходу движения: `rb.AddForce(transform.right * currentThrust, ForceMode.Force);`.
* *Таксировочный бустер (Taxi Assist):* Тяжелому физическому объекту в Unity сложно сдвинуться с места из-за сил трения покоя коллайдеров. Данный блок `if (isGrounded && throttle > 0.02f ...)` добавляет стартовый импульс тяги на ультра-малых скоростях, помогая автожиру начать плавное движение на рулении.


9. **Стабилизация и Руление на земле:**
При нажатии `A/D` на земле к машине прикладывается крутящий момент вокруг вертикальной оси Y: `rb.AddTorque(transform.up * yawInput * yawSensitivity * 2500f * groundTurnModifier)`. Это имитация поворота переднего рулевого колеса. Дополнительно рассчитывается компенсирующий момент против опрокидывания на бок при резком повороте.
10. **Стабилизация и Аэродинамика хвоста в воздухе:**
В воздухе колесо не работает. Поворотами и стабилизацией управляет хвост (оперение).
* `float tailEffectiveness = Mathf.Clamp01(forwardSpeed / 14f);` — эффективность хвоста равна 0 на месте и становится стопроцентной при скорости $14\text{ м/с}$ и выше, когда поток воздуха плотно обдувает киль.
* `rb.AddTorque(transform.up * -sidewaysSpeed * 22f * tailEffectiveness, ForceMode.Force);` — флюгерный эффект стабилизации курса. Если автожир сносит боком (`sidewaysSpeed`), хвост автоматически разворачивает машину носом по ветру.
* Дополнительно блоки гашения паразитных колебаний (демпфирование по тангажу и крену) имитируют сопротивление плоскостей хвоста воздушной среде.
* Руление в воздухе: `rb.AddTorque(transform.up * yawInput * yawSensitivity * 480f * (tailEffectiveness + 0.25f), ForceMode.Force);` — аэродинамический поворот от руля направления.


11. **Поворот визуального киля:**
Объект `rudderKiel` физически поворачивается в локальных координатах на угол до 28 градусов функцией плавного перехода `Quaternion.Slerp`.

### Метод `HandleEngineAudio()`

Реализует последовательную работу звукового движка (конечный автомат).

* Если двигатель запущен:
* Проверяет `if (!wasEngineRunningLastFrame)`. Если в прошлом кадре он спал, а сейчас работает — значит, произошел запуск. Включается `startAudioSource` с клипом стартера, флаг `starterFinished` сбрасывается в `false`.
* Как только одиночный клип стартера доиграл (`!startAudioSource.isPlaying`), флаг `starterFinished` переводится в `true`.
* При `starterFinished == true` активируется постоянный фоновый гул `loopAudioSource`.
* **Динамический питч (Pitch):** `loopAudioSource.pitch = Mathf.Lerp(0.65f, 1.35f, engineRPMPerc);`. Частота звука (высота тона) и его громкость напрямую зависят от текущих оборотов двигателя. При увеличении газа мотор начинает реветь громче и на более высоких частотах.


* Если двигатель выключен: все аудиоисточники принудительно останавливаются (`Stop()`).

---

### Метод `HandleRecovery()` (Эвакуация на клавишу R)

* **Что принимает:** Ничего.
* **Что делает:** Проверяет нажатие клавиши `R`. Если игрок разбился, перевернулся или улетел за пределы карты:
1. Сбрасывает углы крена и тангажа в ноль, оставляя только текущий курс по компасу (`transform.rotation = Quaternion.Euler(0f, transform.rotation.eulerAngles.y, 0f);`). Автожир мгновенно встает на ровный киль.
2. Приподнимает машину на 2 метра вверх над текущей точкой, чтобы она не застряла текстурами в земле (`transform.position += Vector3.up * 2.0f;`).
3. Обнуляет векторы линейных и угловых скоростей Rigidbody, полностью останавливая падение или дикое вращение.
4. Принудительно включает стояночный тормоз (`isParkingBrakeOn = true`), чтобы машина не покатилась под уклон сразу после восстановления.

### Метод `RotateVisualVanes()`

Отвечает за сугубо визуальное вращение пропеллеров в кадре, основываясь на математических расчетах скоростей.

* Каждый кадр к переменной `currentRotorVisualAngle` добавляется значение `rotorRPM * 6f * Time.deltaTime` (перевод оборотов в минуту в градусы в секунду). При достижении $360^\circ$ угол сбрасывается, чтобы не перегружать память регистра.
* **Сложная визуальная трансформация диска ротора:**
```csharp
Quaternion tiltRotation = Quaternion.AngleAxis(currentRotorTiltPitch, Vector3.forward) * Quaternion.AngleAxis(-currentRotorTiltRoll, Vector3.right);
mainRotor.localRotation = tiltRotation * Quaternion.Euler(0f, currentRotorVisualAngle, 0f);

```


Здесь перемножаются три кватерниона. Скрипт сначала наклоняет всю виртуальную втулку ротора вслед за ручкой управления (РУС) игрока по осям Pitch/Roll, и одновременно с этим вращает лопасти по кругу вокруг их собственной локальной оси Y на вычисленный угол `currentRotorVisualAngle`. Это создает идеальную картинку работы настоящего вертолетного/автожирного винта.
* Маршевый винт `pusherPropeller` аналогично вращается вокруг своей оси со скоростью, завязанной на текущие обороты двигателя (`engineRPM`).

---

### Метод `GetAltitude()`

* **Что принимает:** Ничего.
* **Что возвращает:** Значение типа `float`.
* **Что делает:** Возвращает текущую мировую координату автожира по высоте (`transform.position.y`). Используется HUD-интерфейсом (высотомером) для вывода цифр на экран пользователя.
